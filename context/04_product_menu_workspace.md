# 💻 [Context 4] 메뉴별 화면/업무 컨텍스트 문서 (Menu Workspace)

본 문서는 Streamlit 웹 플랫폼(`app.py`)으로 구동되는 **Automotive OE Requirements Explorer**의 5대 핵심 메뉴에 대한 화면 구성, 기능 구성, 그리고 사용자의 실제 업무 시나리오(Workflows)를 상세히 설명한 메뉴별 업무 컨텍스트 가이드입니다.

---

## 🎨 대시보드 화면 및 UI 테마 디자인 핵심 가이드

본 시스템의 UI/UX 디자인은 사용자의 시각적 편의를 극대화하고 고품격 전문가용 플랫폼의 이미지를 심어주기 위해 다음과 같은 규칙에 따라 구현되었습니다:

*   **테마 및 색상 시스템 (HSL Palette)**:
    - 주조색: 깊고 신뢰감을 주는 슬레이트 다크 블루 (Slate Dark Blue) 및 프리미엄 글래스모피즘(Glassmorphism) 배경.
    - 강조색: 전자기기 계기판 느낌의 테크니컬 시안 블루 (`#00c8ff`) 및 웜 실버.
    - 리스크 경고색: 리스크 수치에 따라 동적으로 노출되는 소프트 레디시 엠버 및 브라이트 레드.
*   **컴포넌트 레이아웃**:
    - 사이드바(Sidebar)를 공통 컨트롤 타워로 삼아 고객사 및 공장, 공정 필터를 실시간 조정합니다.
    - 메인 영역은 시각 지표(Top) ➔ 메인 표 및 결과(Middle) ➔ 세부 정보 및 다운로드(Bottom)로 일관된 시각 흐름을 따릅니다.

## 📊 1. [메뉴 01] Risk Assessment (공장 현장 리스크 분석 및 경고)

### ① 화면 레이아웃 (Layout)
*   **리스크 요약 KPI 카드 영역 (Top Index Cards)**:
    - 4개의 프리미엄 글래스모피즘 카드: **총 누적 리스크 건수 (Total Risks)**, **미해결 품질 이슈 (Open QI)**, **미점검 4M 변경점 (Pending 4M Changes)**, **과거 감사 미해결 지적사항 (Open Audit Findings)**이 실시간으로 상단에 표시됩니다.
*   **인터랙티브 리스크 맵 & 차트 영역**:
    - **공장별 실시간 위험 점수(Plant Risk Score, Max 5.0)**를 Plotly 동적 도넛 차트 및 가로 바 차트로 시각화합니다.
    - 가류(Curing), 성형(Building) 등 취약 고위험 공정의 리스크 누적 비율 분포 차트를 제공합니다.
*   **하단 공장별 고위험 실시간 경고판 (Live Risk Alert Board)**:
    - 특정 공장에서 품질 실패(QI), 4M 변경점, 과거 지적사항 중 임계값(3.5점 이상)을 초과하는 리스크 항목을 감지하면, **"⚠️ DP 공장 가류공정 긴급 수검 대비 권장 (과거 실패 이력 존재)"**과 같은 리스크 요약 경고 마이크로 마커를 실시간 출력합니다.

### ② 화면 기능 구성 (Features)
*   **실시간 다이내믹 가중치 연산**: DB 내 과거 1년간 품질 이슈 빈도 및 변경 빈도를 기반으로 공장 리스크 가중치를 실시간 자동 재계산합니다.
*   **인터랙티브 툴팁 및 경고 패널**: 차트 요소에 마우스를 올리면 리스크 기여도를 세부 항목 단위로 분해하여 보여줍니다.

### ③ 사용자의 실무 활용 흐름 (Workflow)
```
대시보드 진입 ➔ 상단 KPI 카드로 전사 리스크 유효율 점검 ➔ 공장별 리스크 차트 분석 ➔ 3.5점 이상 취약 공정(예: 대전 가류공정)에 대한 사전 수검 준비 필요성 조기 진단
```

## 📁 2. [메뉴 02] OE Requirement Library (완성차 규격 요구사항 라이브러리)

### ① 화면 레이아웃 (Layout)
*   **좌측 다중 필터 사이드바**: 완성차 고객사(OEM), 규격 종류(품질 표준, 테스트 시험 규격, 기술 가이드 등)를 선별할 수 있는 다중 셀렉터 배치.
*   **우측 규격서 마스터 그리드**:
    - `id`, `customer`, `doc_code`, `doc_name`, `revision_date`, `doc_type` 등으로 즉각 정렬 가능한 테이블 뷰.
*   **하단 AI 핵심 검토 요약 패널 (AI Review Summary & Document Download)**:
    - 특정 규격서 행을 클릭하면 하단 패널에 **AI가 도출한 핵심 검토 요약본(Review Summary)**이 가독성 높게 출력됩니다.
    - 요약 패널 우측 상단에 해당 완제품 규격 원본 파일을 로컬로 직접 받을 수 있는 **[규격 원본 다운로드 (Download File)]** 고성능 버튼을 제공합니다. (체크리스트 질문과는 완전히 별개로 독립되어 운영됨).

### ② 화면 기능 구성 (Features)
*   **규격 요약 4대 지표 자동 추출**: 적용 범위, 정기 재자격 부여 주기, 특별 특성 지정 의무, 품질 사고 발생 시 긴급 보고체계 요약을 고속 렌더링합니다.
*   **로컬 보안 파일 다운로드 트리거**: 관리자가 적재한 원본 문서를 유실 없이 안전하게 브라우저로 내려받을 수 있도록 파일 전송 프로토콜을 구현합니다.

### ③ 사용자의 실무 활용 흐름 (Workflow)
```
고객사(예: BMW) 필터링 ➔ 최근 개정된 규격 문서('GS 91011') 선택 ➔ 하단 패널에서 AI 핵심 요약본을 신속히 스캔 ➔ 증빙 실사 대조를 위해 '오리지널 PDF 규격서' 다운로드 클릭
```

## 📋 3. [메뉴 03] Self-Audit & Master Checklist (자체 감사 및 마스터 체크리스트)

### ① 화면 레이아웃 (Layout)
*   **최상단 3중 마스터 필터 컨트롤러**:
    - **공장별 필터 (Plant Selector)**: ALL(전사 공통) 또는 개별 공장 코드(`DP`, `KP`, `MP` 등) 선택.
    - **OEM별 필터 (OEM Selector)**: `BMW`, `Audi`, `Hyundai`, `GM` 등 특정 완성차 고객사 선택.
    - **공정별 필터 (Process Selector)**: `Incoming`, `Mixing`, `Curing`, `Building` 등 15대 표준 공정 선택.
*   **중단 통합 검색 및 세부 제어바**:
    - 원천 구분 필터 (`DOCUMENT`(규격서 기반), `DATABASE_QI`(실패 기반), `DATABASE_4M`(변경점 기반), `DATABASE_AUDIT`(지적사항 기반)) 선택.
    - 실시간 텍스트 검색바 (`audit_question` 및 `requirement` 내부 단어 직접 매칭).
*   **하단 통합 마스터 체크리스트 그리드**:
    - 필터링된 모든 질문, 요구 조항, 수검 대응 증적, 감사 방법이 상세 테이블로 출력되며, **우측 상단에 [CSV 다운로드] 단추 배치**.

### ② 화면 기능 구성 (Features)
*   **한글 깨짐 방지 UTF-8 BOM (UTF-8-SIG) 내보내기**: 한국어 윈도우 엑셀 환경에서도 한글이 전혀 깨지지 않고 정상 인코딩되어 즉시 열리는 CSV Export 기능을 완벽 제공합니다.
*   **동적 우선순위 뱃지 시스템 (Priority Badges)**: 리스크 가중치가 높은 항목은 붉은색 `High` 뱃지, 보통 항목은 황색 `Medium` 뱃지가 동적으로 깜빡이며 가독성을 보조합니다.

### ③ 사용자의 실무 활용 흐름 (Workflow)
```
공장 필터 'MP(헝가리)' + OEM 필터 'BMW' + 공정 필터 'Curing(가류)' 지정 ➔ BMW 규격 및 과거 헝가리 공장의 이력이 통합된 자체 감사 체크리스트 질문 실시간 선별 ➔ CSV 다운로드 ➔ 현장 순회 점검표로 인쇄하여 활용
```

## 🔍 4. [메뉴 04] AI Action Plan (AI 조치 계획 및 개선 가이드)

### ① 화면 레이아웃 (Layout)
*   **상단 심사 질문 연동 카드**:
    - 자체 수검 또는 외부 오디트 중 발견된 지적 항목이나 검토 대상 마스터 체크리스트 항목을 원클릭 선택하거나 직접 부적합 상황을 입력할 수 있는 텍스트 아레아 패널 제공.
*   **중단 AI 피드백 제안 트리거**:
    - **[AI 개선 조치 가이드라인 생성 (Generate AI Action Plan)]** 버튼 배치.
*   **하단 AI 종합 피드백 보드 (AI Action Feed)**:
    - 제출 즉시 AI 모델이 대화형 글래스모피즘 보드 내에 해당 질문/지적 사항의 리스크 제거를 위한 **영구 조치 계획(Corrective Action)**, **작업 표준서(SOP) 개정 권고 내용**, 완성도 높은 **현장 실물 증적(Evidence) 보완 가이드**를 체계적으로 출력합니다.

### ② 화면 기능 구성 (Features)
*   **Gemini AI 연동 프롬프트 엔지니어링**: 자동차 산업 품질 감사 표준인 VDA 6.3 가이드라인 및 IATF 16949 영구 시정 대책 수립 프로세스(8D Report 형식)에 맞추어 보완 행동 가이드를 동적으로 조합해 냅니다.
*   **SOP 매핑 피드백**: 부적합 개선을 위해 수정해야 할 실제 공정 SOP 파일의 구체적 조항 및 검교정 점검 주기 개정 방안을 권장 항목으로 보여줍니다.

### ③ 사용자의 실무 활용 흐름 (Workflow)
```
실제 OEM 감사 중 '가류공정 압력 센서 검교정 주기 미준수' 지적사항 발생 ➔ AI Action Plan 메뉴에 해당 내용을 체크리스트 연동 입력 ➔ AI가 제안하는 '압력 센서 SOP 교정 주기 변경안 및 교정성적서 백업 일치성 검증 행동 지침' 즉각 수립 ➔ 협력사 조치 계획서(Action Plan) 초안으로 즉시 채택 및 활용
```

## 💻 5. [메뉴 05] SQL Console (고급 사용자용 실시간 SQL 콘솔)

### ① 화면 레이아웃 (Layout)
*   **상단 SQL 쿼리 템플릿 셀렉터 (Template Loader)**:
    - "자주 쓰는 쿼리 템플릿"을 원클릭으로 선택할 수 있는 간편 서식 드롭다운 제공. (예: 공장별 리스크 점수 통계, 특정 고객사의 중요도 High 체크리스트 조회, 테이블 조인 분석 등).
*   **중단 SQL 쿼리 텍스트 영역 (Interactive Code Editor)**:
    - 다중 라인 입력이 가능한 대형 쿼리 입력 에디터 창 및 **[쿼리 실행 (Run Query)]** 버튼.
*   **하단 실행 결과 그리드 및 다운로드**:
    - 실행 완료 시 소요 시간과 출력된 행(Rows) 수가 표시되며, 쿼리 결과를 즉시 CSV로 다운로드할 수 있는 버튼 배치.
*   **하단 안전 경고 영역**:
    - 데이터 변경을 감지하고 차단할 시 표시되는 선명한 안내문 영역.

### ② 화면 기능 구성 (Features)
*   **안전성 차단 메커니즘 (Security Sandbox Filter)**:
    - 비독점적/악의적 쿼리를 통한 데이터 훼손을 차단하기 위해 `SELECT` 문을 제외한 모든 쓰기/변경 키워드(`INSERT`, `UPDATE`, `DELETE`, `DROP`, `ALTER`, `CREATE`, `REPLACE` 등)를 사전에 대소문자 구분 없이 감지하여 쿼리 실행을 완벽히 차단하고 경고 창을 띄웁니다.
*   **자동 에러 바운더리(Exception Handler)**: SQL 문법 오류 또는 잘못된 컬럼 쿼리 시, 프로그램이 크래시되지 않고 하단에 에러 문구를 텍스트로 친절하게 출력해 줍니다.

### ③ 사용자의 실무 활용 흐름 (Workflow)
```
SQL 콘솔 메뉴 진입 ➔ 템플릿 드롭다운에서 '공장별 리스크 점수 통계' 로드 ➔ 실행 버튼 클릭 ➔ 반환된 분석 결과를 데이터프레임으로 검증 및 엑셀 가공용 CSV 다운로드
```
