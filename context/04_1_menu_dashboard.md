# 📊 04_1_menu_dashboard

## 1. 개요 (Overview)
본 화면은 완성차 고객사(OEM) 기술 규격 조항과 제조 공정의 품질 실패 이력(QI), 4M 공정 변경점, 감사 지적사항(Audit Findings)을 지능적으로 연계 분석하여 공장별 리스크 상태를 실시간 계량화하는 **HQ 종합 품질 감사 대시보드(Headquarters Unified Quality Audit Dashboard)**입니다. 

기존의 대시보드 최상단 로컬 필터바를 제거하여 **HQ(본사) 종합 모니터링 관점으로 완전히 고정**함으로써, 8대 글로벌 생산 공장의 전체적인 품질 리스크 분포와 취약 공정을 전사적 차원에서 다차원적으로 실시간 감시 및 조망할 수 있도록 아키텍처를 고도화하였습니다.

최근 고도화를 통해 기존의 `Live Risk Alert Board` 섹션을 완전 제거하고, 대시보드에 감사 지식 자산의 축적 현황을 직관적으로 연계 관측할 수 있는 **`통합 감사 지식 라이브러리 (Library) 현황` 위젯을 3번째 섹션으로 신규 추가**하였으며, 기존 상단에 존재하던 **`4대 종합 KPI 통계 카드(KPI Board)`를 대시보드 최하단으로 재배치**하여 화면의 시각적 안정성과 정보 밀도를 극대화하였습니다.

## 3. 핵심 비즈니스 로직 및 연동 규격 (Business Rules & Data Sync)

### ① 4대 종합 KPI 통계 카드 (최하단 배치)
대시보드 KPI 카드는 로컬 `data/` 디렉토리의 정적 데이터베이스(`cqms_qualityissue_db.json`, `cqms_4m_db.json`, `cqms_customer_audit_db.json`)와 100% 무지연 실시간 연동되어 온디맨드로 계산 및 갱신되며, 대시보드 화면 최하단에 배치되어 전사 실시간 위험 및 분포 차트를 확인한 후 최종적인 전사 지표를 요약 감시할 수 있도록 지원합니다.

| KPI 카드명 | 주요 연산 공식 및 뱃지 매핑 규칙 | 수치 데이터 연동 원천 |
| :--- | :--- | :--- |
| **총 누적 리스크 건수** | $\text{총 미결 건수} = \text{QI 미결} + \text{4M 미결} + \text{지적사항 미결}$<br>· 종합 진행률: $(\text{해결 완료 합산} / \text{누적 전체 합산}) \times 100$<br>· UI 구성: 90deg Linear Blue-Cyan 그라데이션 진행률 게이지 바 | · `qualityIssues` (On-going)<br>· `changeHistory4m` (On-going)<br>· `auditFindings` (On-going, Open) |
| **미해결 품질 이슈 (QI)** | · 수량: `STATUS === 'On-going'` 인 품질 실패 건수 총합<br>· 처리율: $(\text{Complete 또는 Closed} / \text{전체 QI}) \times 100$<br>· UI 구성: High-Risk Red 아이콘 배지 및 Red-Coral 그라데이션 처리 게이지 바 | · `cqms_qualityissue_db.json`<br>(STATUS: On-going / Complete) |
| **미점검 4M 변경점** | · 수량: `PROGRESS === 'On-going'` 인 공정 4M 변경 신청 건수 총합<br>· 점검률: $(\text{Complete} / \text{전체 4M}) \times 100$<br>· UI 구성: Medium-Risk Amber 아이콘 및 Amber-Yellow 게이지 바 | · `cqms_4m_db.json`<br>(PROGRESS: On-going / Complete) |
| **과거 감사 미해결 지적** | · 수량: `STATUS === 'On-going' 또는 'Open'` 인 과거 감사 지적사항 건수 총합<br>· 조치율: $(\text{Complete 또는 Closed} / \text{전체 Findings}) \times 100$<br>· UI 구성: High-Risk Purple 아이콘 및 Purple-Violet 게이지 바 | · `cqms_customer_audit_db.json`<br>(STATUS: On-going, Open / Complete, Closed) |

### ② 차트 시각화 명세 (Chart.js Integration)

#### [Chart 1] 공장별 실시간 위험 점수 (Plant Risk Score)
*   **Bar 모드 (기본 뷰)**: 8대 생산 공장('DP', 'KP', 'JP', 'HP', 'CP', 'MP', 'IP', 'TP')의 가중 리스크 스코어($R_P$)를 일괄 비교하는 수직 막대 차트입니다.
    *   **컬러 동적 뱃지 룰**: 위험 스코어가 임계치 $3.5$점 이상일 경우 위험(Red, `rgba(239, 68, 68, 0.75)`) 색상으로 동적 변경되며, 미만일 경우 기본 블루(`rgba(59, 130, 246, 0.75)`) 색상으로 하이라이팅됩니다.
    *   **공식**: $R_{P} = \min \left( 5.0, \;\; 0.3 \times N_{QI}(P) + 0.1 \times N_{4M}(P) + 0.2 \times N_{Audit}(P) \right)$ (미결 unresolved 기준 가중치 연산)
*   **Doughnut 모드 (원인분석 뷰)**: 바 차트의 특정 공장 막대를 클릭 시, 해당 공장의 3대 기여 요인(품질이슈 기여도 점수, 4M 기여도 점수, 지적사항 기여도 점수)을 한눈에 분할 관측하는 도넛 차트로 미려하게 스위칭됩니다.
    *   **목록 복귀 인터랙션**: 차트 제목 우측에 [목록 복귀 ↩] 버튼이 동적으로 생성되어 원클릭 시 원래의 8대 공장 비교 바 차트로 자연스럽게 복귀합니다.

#### [Chart 2] 취약 공정별 리스크 누적 분포 (Vulnerable Process Score)
*   15대 표준 공정 및 4대 시스템 카테고리에 대해 전 공장 종합 리스크 지수를 연산한 뒤, 상위 6대 고위험 취약 공정을 자동으로 추출하여 내림차순 정렬하는 수평 바(Horizontal Bar) 차트입니다.
*   점수대별 컬러 매핑:
    *   $3.5$점 이상: 고위험 경고 레드 (`rgba(239, 68, 68, 0.7)`)
    *   $2.0$점 이상: 중위험 경고 앰버 (`rgba(245, 158, 11, 0.7)`)
    *   $2.0$점 미만: 안전/일반 블루 (`rgba(59, 130, 246, 0.7)`)

### ③ 통합 감사 지식 라이브러리 (Library) 현황 위젯 (메뉴 5 연동)
*   **목적**: 대시보드 3번째 섹션에 위치하며, 메뉴 5(Library) 및 AI 감사 비서가 관리하는 전사 감사 기술 지식 자산의 최신 축적 현황을 직관적인 3대 요약 카드 그리드로 제공합니다.
*   **데이터 연동 규격**:
    *   **고객사(OEM) 기술 요건 문서** (`#db-lib-doc-count`): 전역 상태 `state.documentLibrary` 배열의 총 개수를 바인딩하여 표출합니다.
    *   **표준 감사 마스터 체크리스트** (`#db-lib-checklist-count`): 전역 상태 `state.auditChecklists` 배열의 총 개수를 바인딩하여 표출합니다.
    *   **과거 OE Audit 지적사항 (SOP)** (`#db-lib-findings-count`): 전역 상태 `state.auditFindings` 배열의 총 개수를 바인딩하여 표출합니다.
*   **Aesthetic UI & 마이크로 인터랙션**:
    *   우측 상단에 `[라이브러리 지식 맵 이동 ↩]` 숏컷 배지 버튼을 제공하여, 클릭 시 즉시 메뉴 5(Library) 탭으로 스무스하게 전환되는 프리미엄 내비게이션 환경을 구현했습니다.
    *   각 카드는 은은한 반투명 글래스모피즘(`background: rgba(30, 41, 59, 0.02)`) 스타일과 정밀 보더라인을 갖추고 있으며, 풍부하고 가독성 높은 메타 정보(Hyundai, GM 등 OEM 탑재 현황, 중복 질문 제거 알고리즘 설명, 지능형 자연어 분류 기술 설명 등)를 제공합니다.

### ④ 향후 예정된 Audit 일정 및 준비현황 요약 위젯 (메뉴 2 연동)
*   **목적**: 대시보드 내에서 다가오는 중요 고객사(OEM) 감사 일정을 일목요연하게 트래킹하고, 메뉴 2 (Audit Planning)의 실시간 준비 완료 진척도를 통합 요약하여 오디터에게 직관적인 일감 트래킹 뷰를 선사합니다.
*   **데이터 연동 및 정렬 규칙**:
    *   `state.audits` 목록 중 상태가 `"Open"`인 미래 수검 일정을 날짜(`START_DT` 또는 `date`) 오름차순으로 정렬하여 가장 임박한 수검 예정 카드 상위 3개를 선별 노출합니다.
    *   각 카드는 메뉴 2의 `planningChecklistStates` 및 10대 마일스톤 태스크 데이터셋을 즉석 연산하여 `준비 완료도 %`와 `완료 건수 / 대기 건수` 통계를 실시간으로 동기화합니다.
*   **Aesthetic UI & 마이크로 인터랙션**:
    *   우측 상단에 `[전체 일정 관리 이동 ↩]` 숏컷 배지 버튼을 지원하여, 클릭 시 즉시 메뉴 2(Audit Planning) 탭으로 스무스하게 전환되는 고급 내비게이션 환경을 구현했습니다.
    *   개별 감사 요약 카드는 마우스 호버(Hover) 시 살짝 솟아오르는 인터랙티브 효과(`transform: translateY(-2px); border-color: var(--brand-blue-alpha);`)와 부드러운 트랜지션(0.2s ease-in-out)을 적용해 시각적 유연함을 격상시켰습니다.

### ⑤ 글로벌 공장별 종합 품질 리스크 지수 (CRI) 현황 위젯 (메뉴 3 연동)
*   **목적**: 전사 8대 생산 공장의 종합 품질 리스크 지수(CRI, Composite Risk Index)를 실시간 계산하여 대시보드 내에 시각화(왼쪽 컴팩트 수평 막대그래프) 및 리더보드(오른쪽 4*2 카드 그리드) 2열 구조로 통합 관측할 수 있게 합니다.
*   **레이아웃 구성**:
    *   **왼쪽 열 (CRI 비교 차트)**: Chart.js 기반의 컴팩트 수평 막대그래프(Horizontal Bar Chart)로, 세로 높이를 `270px`로 완벽하게 고정(Fix)하여 8대 생산 공장의 실시간 CRI 스코어를 직관적으로 정량 비교합니다.
    *   **오른쪽 열 (CRI 카드 리더보드)**: 기존 공장별 CRI 리더보드 카드의 개별 높이를 `130px`로 엄격히 고정(Fix)하고 **가로 4열, 세로 2행 (4*2) 구조**로 조밀하게 정렬 배치하여 총 높이 `270px`로 양쪽의 세로 균형을 완벽히 일치시키고 최상위 정보 밀도를 수용합니다.
*   **데이터 연동 및 정렬 규칙**:
    *   메뉴 3 (Plant Risk & Action) 의 리더보드 데이터 및 원천 세부 데이터셋을 상시 스캔 및 즉석 관계 연산하여 `CRI 점수`를 실시간 계산합니다.
    *   CRI 수치 기준에 따라 8대 공장을 리스크가 높은 순(내림차순)으로 자동 정렬하여 차트와 카드 리스트에 실시간 렌더링합니다.
    *   각 카드 하단에는 세부 평가 차원 점수(인프라 구축도 %, 현장실사 합격률 %, 중대 품질이슈 건수)를 동적으로 매핑하여 지수의 구성 원인을 한눈에 분석할 수 있게 돕습니다.
*   **Aesthetic UI & 마이크로 인터랙션**:
    *   **막대 차트**: CRI 리스크 수치에 따라 바의 채우기/테두리 색상이 동적으로 변경됩니다 (CRITICAL: Red `rgba(239,68,68,0.75)`, MODERATE: Amber `rgba(245,158,11,0.75)`, LOW: Green `rgba(16,185,129,0.75)`). 마우스를 바에 오버할 때 팝업되는 고급 툴팁에 인프라/실사/이슈 성분을 분할 기재하여 보여줍니다.
    *   **상호작용 내비게이션**: 막대그래프의 바를 클릭하거나 오른쪽 개별 공장 카드를 클릭하는 즉시, 전역 상태 변수 `plantRiskActivePlant` 및 `selectedPlant`가 해당 공장 코드로 자동 업데이트되며, 동시에 3번 메뉴(Plant Risk & Action) 탭으로의 내비게이션 및 세부 차트/테이블 0.1초 전체 갱신이 완벽하게 동기화되어 즉시 수행됩니다.
    *   **카드 마우스 오버**: 각 카드는 마우스 호버(Hover) 시 세련된 브랜드 블루 보더(`var(--brand-blue)`)와 부드러운 그림자 효과(`boxShadow`), `transform: translateY(-2px)` 트랜지션을 주어 프리미엄 조작 감각을 선사합니다.
    *   **고대비 뱃지**: CRI 수치별 고대비 HSL 리스크 뱃지(CRITICAL, MODERATE, LOW)를 부여하여 오디터의 긴급한 시선 집중을 유도합니다.

## 4. UI/UX 및 웹 품질 표준 준수 (Aesthetics & Accessibility)
1.  **최상단 필터 기능 제거를 통한 화면 청결성 극대화**: 사용자가 화면에 진입하자마자 전사 차원의 위험 분포 상태를 방해 요소(Dropdown, Button 등) 없이 직관적으로 1초 만에 감지할 수 있도록 설계 정밀성을 강화했습니다.
2.  **명도 대비 AA 규격 수렴**: 어두운 슬레이트 카드 내 텍스트 컬러의 명도 대비를 WCAG 2.1 AA 권장값(4.5:1 이상)에 맞춰 정교하게 조율하여, 고령의 제조 공정 실사 및 오디터 분들도 장시간 무리 없이 탐색 가능하도록 가독성을 극대화했습니다.
3.  **성능 최적화**: 차트 전환 시 기존 Chart.js 인스턴스를 메모리에서 즉각 완전히 파괴(`destroy()`)하고 재성성하여, 싱글 페이지 프론트엔드(SPA) 아키텍처 상의 잔상 렉이나 메모리 누수(Memory Leak)를 완전히 방지했습니다.
