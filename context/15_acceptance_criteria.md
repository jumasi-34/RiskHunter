# ✅ 15 Acceptance Criteria (인수 완료 조건서)

본 문서는 **Risk-Based Audit Checklist System** 사내 해커톤 MVP 플랫폼이 최종 릴리즈 및 시연을 통과하기 위해 완벽히 준수해야 할 **품질 보증 및 기술/Aesthetic 인수 완료 요건(Exit Criteria)**을 기술합니다. 

모든 마일스톤 페이즈가 마감될 때마다 본 기준서를 기반으로 교차 검증을 수행하며, 단 하나의 요건이라도 미충족될 시 해당 버전은 배포 승인이 불가합니다.

---

## 🎨 1. 웹 접근성 및 명도 대비 무결성 검증 (Contrast & Legibility)

시연 당일의 빔프로젝터 환경이나 조명 여하에 관계없이 모든 화면에서 텍스트와 보조 지표가 칼처럼 선명하게 정독되어야 합니다.

*   **밝은 배경 영역 (메인 캔버스, 화이트 카드 컴포넌트)**:
    *   카드의 보조 텍스트, 설명문, 통계 단위, 테이블의 비활성 라벨 등은 반드시 **`--text-muted-light: #64748b`** (또는 그 이상의 명도 대비 텍스트 컬러)가 매핑되어야 합니다.
    *   **인수 기준**: 순백색 `#ffffff` 배경과의 명도 대비율이 최소 **4.5:1 이상**일 것. (Lighthouse 진단에서 명도 대비 관련 경고가 0건일 것)
    *   절대 어두운 배경용 `--text-muted-dark: #94a3b8`을 밝은 카드 영역에 적용하는 오류를 범하지 않을 것.
*   **어두운 배경 영역 (사이드바 메뉴, SQL 콘솔)**:
    *   메뉴 라벨 및 가이드 텍스트는 **`--text-muted-dark: #94a3b8`** 또는 **`--text-light: #ffffff`**가 장착되어야 합니다.
    *   **인수 기준**: 어두운 배경 `#0f172a`와의 대비율이 **4.5:1 이상**일 것.
*   **자가 검증 방법**:
    1. Chrome 브라우저에서 `F12`를 눌러 개발자 도구를 엽니다.
    2. 검사(Inspect) 도구를 선택하고 화면의 임의의 작은 회색 글씨(예: D-Day 서브 타이틀, KPI 단위, 테이블의 기한 표시 등) 위에 마우스를 가져갑니다.
    3. 툴팁에 표시되는 **`Contrast` 항목에 초록색 더블 체크마크(AA)가 표시되는지 확인**합니다.

## 🔄 2. SPA 구조 및 탭 네비게이션 검증 (SPA Lifecycle & Tab Stability)

*   **바닐라 탭 전환 지연 제로**:
    *   메뉴(대시보드, 규격서, 체크리스트, AI 액션, SQL 콘솔) 탭 클릭 시, 페이지 전체가 새로고침되거나 흰 화면(White-out)이 번쩍이는 현상이 없어야 합니다.
    *   **인수 기준**: 이전 활성 탭이 숨겨지고 타겟 탭이 `.active` 클래스와 함께 부드러운 이징 효과(Fade-In, Slide-Up 트랜지션)를 동반하며 즉각(0.2s 내) 교체 렌더링될 것.
*   **사이드바 활성 상태 동기화**:
    *   탭이 전환될 때 좌측 사이드바의 탭 메뉴에 로열 블루 배경의 `active` 클래스가 정확히 이식되어 사용자에게 시각적 동조감을 선사할 것.
*   **타겟 외 메뉴 가이드 보존**:
    *   현재 페이즈 이외의 비타겟 탭 메뉴(예: Admin Settings 등)는 완전히 지우지 않고, 글래스모피즘 스켈레톤 레이아웃이나 "개발 대기 중 (Phase 7 예정)" 안내 패널이 품위 있고 아름답게 잔존해 있어야 할 것.
*   **가변 너비 레이아웃 붕괴 검증 (Squeeze & Overflow Test)**:
    *   **인수 기준**: 브라우저 가로 폭을 최저 1280px 해상도까지 축소하여 압착 테스트를 진행했을 때, 글로벌 필터 초기화(Reset) 단추 등 좁은 영역의 액션 버튼 내 텍스트가 아래 줄로 튕기거나(Wrapping) 버튼 바깥 영역으로 글자가 튀어나와 겹쳐 보이는 시각 오동작(Overflow)이 단 1건도 발견되지 않을 것.
    *   **자가 검증 방법**: 브라우저 창 우측 경계를 드래그하여 점진적으로 축소하면서 초기화 단추가 온전한 정적 너비를 고수한 채 글자가 찌그러지지 않고 단정하게 한 줄을 유지하는지 수동 검정할 것.

## 🧮 3. 실시간 리스크 연산 및 가중치 데이터 정합성 검증 (Risk Matrix Integrity)

*   **동적 리스크 점수 합산 공식 준수**:
    *   공장별 공정 카테고리의 리스크 가중치 산출 공식 (4-Way):
        $$\text{Plant Risk Score} = \text{Clamp}\Big( (\text{QI Count} \times 0.3) + (\text{4M Change Count} \times 0.1) + (\text{Audit Findings Count} \times 0.2) + (\text{Internal Audit Count} \times 0.15), \; 0.0, \; 5.0 \Big)$$
    *   위 공식이 자바스크립트 상에서 소수점 첫째 자리까지 정확하게 계산되고 필터 변경 시 무지연(Real-time) 리렌더링될 것.
*   **위험도 연계 동적 뱃지 노출**:
    *   도출된 리스크 점수가 **3.5점**을 초과하는 행이나 카드는 강렬한 경고 뱃지(`High` / Red)와 보더라인 하이라이트가 누락 없이 트리거될 것.
*   **차트 동적 갱신**:
    *   공장, OEM 필터 조작 시 차트의 데이터 계열(Datasets)과 비례 도넛 아크(Arc Gauge)가 깜빡임 없이 유려하게 크기 변화하여 싱크될 것.

## 🛡️ 4. SQL 샌드박스 콘솔 보안 및 예외 제어 검증 (SQL Safe Sandbox Execution)

*   **안전 SELECT 샌드박스 작동**:
    *   사용자가 SQL 쿼리 입력창에 직접 쿼리를 쓰거나 템플릿을 실행할 때, 오직 단순 SELECT 조회 연산만 메모리에서 매핑 구동되어 데이터 프레임 뷰어로 하단에 노출될 것.
*   **파괴성 쿼리 원천 가로채기(Write Block)**:
    *   사용자가 `INSERT`, `UPDATE`, `DELETE`, `DROP`, `ALTER`, `CREATE`, `REPLACE` 등 원천 정적 데이터를 변형시킬 우려가 있는 키워드를 대소문자 무관하게 하나라도 기입하고 실행을 누를 시, 실제 브라우저가 예외 폭탄을 던지며 먹통이 되지 않고 정규식 검사를 통해 즉시 연산이 차단될 것.
    *   **인수 기준**: 차단과 동시에 "SELECT 전용 안전 샌드박스 상태입니다. 원천 마스터 데이터의 보존을 위해 쓰기 행위는 전면 금지됩니다"라는 우아한 적색 경고 모달 레이어가 정중앙에 수려하게 페이드인될 것.

## 📥 5. CSV 다운로드 UTF-8 BOM 한글 깨짐 무결성 검증 (CSV Export & Byte Inspection)

*   **Excel 즉시 로딩 한글 보존**:
    *   현장 오디터가 체크리스트를 CSV로 다운로드한 후, 국내외 오피스용 MS Excel 프로그램으로 더블클릭 기동하였을 때 한글 질문 및 텍스트가 완전히 깨지는 현상이 없어야 합니다.
    *   **인수 기준**: 다운로드되는 파일의 이진 버퍼 최상두에 **UTF-8 BOM 마커인 `\uFEFF`**가 완벽하게 인젝션되어 엑셀이 인코딩 형식을 즉시 UTF-8로 오토매칭하여 한글을 선명하게 로드할 것.

---

## ⚠️ 6. 무장애 에러 바운더리 및 폴백 레이아웃 검증 (Fallback UI Verification)

*   **White-out 침묵 방지**:
    *   정적 데이터 `/data` 내의 JSON 파일을 고의로 숨기거나 네트워크 차단 상태를 연출하더라도, 화면이 백색으로 기절한 채 사용자 입력을 먹통으로 만드는 현상이 없어야 합니다.
    *   **인수 기준**: 에러 감지 즉시 중앙 캔버스 영역에 "정적 리소스를 로딩할 수 없습니다. 로컬 data/ 폴더 및 파일명을 확인해 주십시오"라는 한글 폴백 카드 레이아웃이 미려하게 노출되어 시연 환경의 무장애 예외 대응 수준을 한눈에 입증할 것.
