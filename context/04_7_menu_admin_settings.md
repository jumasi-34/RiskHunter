# ⚙️ [Context 04_7] Admin Settings (통합 관리자 센터 및 감사 추적 이력 화면 설계서)

본 문서는 Lead Auditor(ADMIN) 권한 활성화 시 진입할 수 있는 **[Admin Settings (관리자 센터)]** 메뉴의 구성 및 화면 동작 요건을 정의합니다. 본 화면은 사용자의 직관적인 사용자 관리, 역할 권한 매트릭스 제어, 그리고 완벽한 ISO/IATF 규격 요구사항 합치성을 증명하기 위한 실시간 시스템 감사 로그(Audit Log)를 제공합니다.

---

## 👥 1. [Sub-Tab 1] User Management (사용자 계정 관리)

### ① 화면 레이아웃 (Layout)
*   **상단 사용자 요약 지표 (User Summary Metrics Row)**:
    *   `총 등록 사용자 수 (3명)`, `활성 오디터 수 (1명)`, `현재 세션 계정 (박정호 수석)`을 보여주는 심플한 글래스모피즘 미니 카드 3개 배치.
*   **중단 사용자 리스트 그리드 (User Registry Table)**:
    *   `data/users.json` 마스터 데이터를 바인딩하여 이름, 소속 부서, 역할 등급(ADMIN/MANAGER/VIEWER), 상태 뱃지, 최종 접속 시간 표시.
    *   우측 끝에 계정 프로필 세부 편집 및 **[사용자 전환 (Simulate Session)]** 퀵 버튼을 두어 시연 중 역할 전환 편의성 극대화.

### ② 화면 기능 구성 (Features)
*   **가상 세션 스위처**: 사용자 행의 [전환] 클릭 시, 사이드바 프로필과 연동되어 실시간으로 세션 역할(Role)이 즉시 변경되며 UI 접근 권한이 자동 재구동됩니다.
*   **신규 사용자 등록 모달 (Add User Modal)**: 이름, 부서, 역할을 지정하여 가상으로 등록하면 `localStorage` 내 사용자 테이블에 즉각 추가되어 테이블에 실시간 렌더링됩니다.

---

## 🔐 2. [Sub-Tab 2] Role & Permission (역할 및 권한 맵)

### ① 화면 레이아웃 (Layout)
*   **인터랙티브 권한 맵 매트릭스 (Role-Permission Matrix Grid)**:
    *   행(Row)에 시스템 주요 기능군(대시보드 수정, AI 개선안 도출, SQL 실행, 체크리스트 승인 등)을 배치하고, 열(Column)에 3대 권한(`ADMIN`, `MANAGER`, `VIEWER`)을 매핑한 수려한 테이블.
    *   권한 보유 여부에 따라 빛나는 녹색 체크 마크(`O`) 및 비활성 회색 대시 마크(`-`)가 표시됩니다.

### ② 화면 기능 구성 (Features)
*   **권한 오버레이 설명**: 특정 체크 마크에 마우스 오버(Hover) 시, 해당 권한이 보장하는 가상 API 및 접근 불가 제한 요소를 우아한 툴팁 효과로 노출하여 심사위원의 시각적 몰입감을 배가시킵니다.

---

## 📝 3. [Sub-Tab 3] Audit Log (감사 및 실무 변경 이력 추적)

### ① 화면 레이아웃 (Layout)
*   **상단 실시간 보안 로그 통계 (Security Log Summary)**:
    *   `총 발생 로그 수`, `보안 이벤트 (SQL 인젝션 차단 등) 수`, `최종 로그 기록 시각` 표시.
*   **중단 감사 이력 타임라인 리스트 (Chronological Audit Trail)**:
    *   터미널 콘솔 감성의 다크 글래스모피즘 카드 내부에 시스템 내에서 발생한 모든 동작(Action)이 시간 역순으로 실시간 적재 출력됩니다.
    *   **로그 데이터 예시**:
        *   `[2026-05-28 14:01:05] [ADMIN: 박정호 수석] SQL Console에서 'SELECT * FROM quality_issues' 쿼리를 정상 실행하였습니다.`
        *   `[2026-05-28 14:02:12] [MANAGER: 이현우 책임] DP 공정 BMW 정기 Audit 일정을 신규 등록 완료하였습니다.`
        *   `[2026-05-28 14:03:55] [SYSTEM: Security Sandbox] 비인도적 SQL 구문(DELETE) 입력을 원천 차단하고 보안 모달을 표시하였습니다.`

### ② 화면 기능 구성 (Features)
*   **실시간 로그 적재 (Live Logger)**: 사용자가 플랫폼 내에서 클릭, 등록, 수정, SQL 실행, 보안 차단 등의 액션을 취할 때마다, 전역 로거 함수(`app.logAction()`)가 트리거되어 `localStorage` 로그 배열에 적재하고 타임라인에 무지연 실시간 추가 렌더링됩니다.
*   **필터 및 검색**: 검색어 입력 시 특정 사용자명이나 행동 키워드로 로그가 고속 필터링됩니다.

---

## 💻 4. [Sub-Tab 4] SQL Explorer (관리자 전용 콘솔)

### ① 화면 레이아웃 (Layout)
*   자주 쓰는 필수 SELECT 쿼리 템플릿 로더 드롭다운 제공.
*   다중 라인 SQL 코드 에디터 패널 및 실행 결과 테이블 렌더링 프레임.
*   하단 빨간색 **[SELECT 전용 안전 샌드박스 가동 중]** 보안 안내 패널 상시 노출.

### ② 화면 기능 구성 (Features)
*   **파괴성 명령어 패턴 스캔**: 에디터에 `INSERT`, `UPDATE`, `DELETE`, `DROP`, `ALTER`, `CREATE` 등이 기입되면, 대소문자 무관하게 감지하여 실행을 원천 차단하고 **"안전 샌드박스에 의해 원천 마스터 데이터 보호가 작동되었습니다."**라는 적색 모달 창을 띄웁니다.
*   **가상 쿼리 프레임**: 클라이언트 메모리 단에서 마스터 JSON 자원을 매핑하여 결과를 동적 테이블로 시각화.
