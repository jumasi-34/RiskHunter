# GEMINI.md

## 1. 프로젝트 정체성
**Risk-Based Audit Checklist System**은 완성차 고객사(OEM) 기술 규격 조항과 공장의 과거 리스크 이력(품질 실패 QI, 4M 공정 변경점, 감사 지적사항 Audit Findings)을 기하학적으로 연계 분석하여 최적의 감사 체크리스트와 AI 조치 대응 계획을 도출하는 **차세대 품질 감사(Audit) 지원 프리미엄 웹 대시보드 시스템**입니다. 
사내 해커톤 MVP 규격에 맞추어 브라우저 온디맨드 구동이 가능하도록 **HTML + CSS + Vanilla JavaScript 기반 고성능 정적 프론트엔드 아키텍처**로 구현됩니다. (상세 비즈니스 개요는 [context/01_product_system_overview.md](file:///home/jumasi/risk_hunter/context/01_product_system_overview.md)를 참고하십시오.)

## 2. 핵심 작업 원칙
*   **전체 시스템을 한 번에 구현하지 않는다.** (거대하고 제어하기 힘든 비즈니스 로직을 한 번에 작성하다 마주하는 스파게티 패턴과 브라우저 에러를 원천 배제합니다.)
*   **반드시 Phase 단위로 작업한다.** (정해진 빌드 마일스톤에 의거해, 이전 페이즈의 정합성과 작동을 수작업 검증한 연후에만 다음 페이즈로 발걸음을 옮깁니다.)
*   **현재 요청한 Phase 외 기능은 구현하지 않는다.** (요청되지 않은 메뉴나 비즈니스 가중치 로직을 임의로 앞당겨 코딩하는 등의 탈선을 원천 금지합니다.)
*   **대상 외 메뉴는 placeholder로 유지한다.** (현재 구현 대상이 아닌 비타겟 탭 메뉴들은 삭제하지 않고, 글래스모피즘 스켈레톤 레이아웃이나 "개발 대기 중"을 나타내는 안내문으로 수려하게 보존합니다.)
*   **관련 없는 파일을 수정하지 않는다.** (타겟 페이즈의 과제 조치와 전적으로 무관한 영역의 소스코드나 기 수립된 마스터 자원 문서를 무분별하게 변경하지 않습니다.)
*   **완료된 Phase를 불필요하게 리팩터링하지 않는다.** (이미 검증을 완료하고 가동 중인 타겟 이전 마일스톤의 무결한 아키텍처를 자기만족적인 기준으로 전면 재작업하는 악순환을 배제합니다.)
*   **복잡한 구조보다 해커톤 데모 안정성을 우선한다.** (시연 당일의 무결한 오동작 방지, 눈길을 즉시 사로잡는 시각적 수려함(Aesthetic WOW) 및 데이터 일치성을 아키텍처의 확장성보다 압도적 상위 우선순위로 간주합니다.)

## 3. 기술 방향 (권장 기술 및 채택 기술)
*   **HTML5**: 레이아웃 컴포넌트의 의미론적 마크업(Semantic Markup) 및 구조화.
*   **Vanilla CSS3**: HSL 글로벌 컬러 변수 시스템, Flexbox/Grid 중심 정렬, 글래스모피즘(Glassmorphism) 프리미엄 효과 및 자연스러운 마이크로 트랜지션.
*   **Vanilla JavaScript**: 비동기 데이터 fetch, 상태 및 사이드바 공통 필터 제어, 클라이언트 메모리 기반 무지연 고속 데이터 가공 및 동적 DOM 렌더링.
*   **Static JSON / CSV data**: `data/` 디렉토리에 정적으로 사전 정제 적재된 데이터 셋.
*   **Browser-side filtering and sorting**: 수만 건 미만의 가볍고 안정적인 데이터 특성을 반영한 자바스크립트 내장 배열 제어 및 고속 필터링 연산.
*   **Chart.js / Plotly.js**: 실시간 리스크 차트 구현을 위해 필요한 단 한 장의 검증된 CDN 로드 차트 컴포넌트.
*   **Mock AI response**: LLM API 호출 유실이나 오프라인 시연 환경에서도 완벽히 사실적이고 전문적인 감사 가이드를 즉시 출력하는 정밀 가상 응답 딕셔너리 설계.

2.  현재 상정된 대상 페이즈(Phase)에 정확히 할당되어 매핑된 최소 참조 문서군 목록만을 엄격하게 골라 집중 독파합니다.

4.  문서 기재 내용 간 충돌이나 비즈니스 가중치 기준 모호성이 관측될 시, 최상위 제어 지휘서인 **00번 인덱스 문서** 및 현재 수행 중인 **해당 페이즈 전용 상세 문서**의 공식 선언을 최우선적인 정답으로 규정합니다.
5.  구현의 깊이나 범위 타협에 애매한 지점이 식별될 시, 해커톤 경계 설계서인 `context/10_mvp_scope_and_non_goals.md`와 자가 인수 검증서인 `context/15_acceptance_criteria.md`를 바탕으로 심플하고 직관적이며 견고하게 종결합니다.

## 5. File Modification Rules
*   **`index.html`**: 전체 화면의 골격을 구성하는 모던 시멘틱 HTML 마크업 제어에 엄격히 집중하며, 이 파일 내부에 자바스크립트 복잡 로직이나 대형 CSS 구문을 몰아서 인라인 코딩하는 스파게티 행위를 전격 금지합니다.
*   **`styles.css`**: HSL 컬러 변수 표준 토큰 선언, 그리드/플렉스박스 구조 설계, 모던 글래스모피즘 효과, 스크롤바 디자인 및 모바일 반응형 뷰포트 배치 등 스타일 정밀화에만 전념합니다.
*   **`app.js`**: 비동기 데이터 fetch 엔진 장착, 글로벌 셀렉터 상태 동적 보존, 공장 리스크 가중치 연산 실행, 실시간 데이터 필터링 및 고속 클라이언트 렌더링 핸들러만 배타적으로 탑재합니다.
*   **`data/` 폴더**: 모의 데이터로 공급되는 깨끗하고 검증된 정적 `JSON` 및 `CSV` 마스터 데이터 파일만 저장할 수 있으며, 실행 코드는 절대 침투하지 못합니다.
*   **`context/` 폴더**: 오직 프로젝트 가이드라인 마크다운 문서만 수납할 수 있습니다.
*   한 번의 코드 실행 턴(Turn) 내에 수십 개의 멀리 떨어진 파일을 동시 다발적으로 마구잡이 수정하는 무리한 소스 제어 행위를 원천 금지합니다.
*   현재 처리 대상이 아닌 타겟 페이즈 이외 파일에 대한 편집 및 수정(Write) 작업은 완벽하게 엄단합니다.

## 6. Phase 작업 규칙
Gemini 에이전트는 전체 시스템을 한 번에 대량 구현하지 않으며, 오직 현재 지시받은 단 하나의 페이즈(Phase) 범위만 순차적으로 집필합니다.

*   **단계별 상세 정의 참조**: 빌드 로드맵 상의 8대 페이즈(Phase 0 ~ Phase 7)별 상세 개발 범위, 산출물 및 인수 완료 요건(Exit Criteria)은 프로젝트 지휘 파일인 **[context/00_context_index_and_build_order.md#6-phase-overview](file:///home/jumasi/risk_hunter/context/00_context_index_and_build_order.md#6-phase-overview)**의 공식 명세를 절대적으로 기준 삼아 행동합니다.
*   **동시 페이즈 개발 절대 금지**: 현재 진행 중인 페이즈의 완료 요건(Exit Criteria)이 완벽히 자체 검증되기 전까지는 다음 페이즈의 파일을 미리 생성하거나 비즈니스 로직을 구현해서는 안 됩니다.

## 7. UI/UX Rules
*   **전문가 테크니컬 디자인**: 완성차 품질 보증원 및 제조 공정 오디터들이 현장 실사 및 오디팅 대기 시 조작하는 계기판 감성의 깊이 있고 고도화된 레이아웃 기법을 채택합니다.
*   **HSL 컬러 가이드**: 슬레이트 다크 블루 및 프리미엄 글래스모피즘(Glassmorphic Translucency)을 심미적 기조로 정립하여 장시간 탐색에도 피로하지 않으면서 최상위 하이엔드 테크 감각을 선사합니다.
*   **정보 가독성의 극대화**: 수십에서 수백 조항에 달하는 텍스트 질문과 대응 증적을 다루는 시스템 특성에 착안하여, 테이블 행 간의 간격과 텍스트의 크기 및 패딩(Padding) 정렬을 일관되게 규율합니다.
*   **고위험 동적 강조**: 연산 공식에 의해 도출된 리스크 점수가 3.5점을 상회하는 위험 요인 및 질문 행은 경고 아이콘과 적색/황색 뱃지를 통해 시야를 강력하게 유도합니다.
*   **자연스러운 마이크로 트랜지션**: 마우스 오버(Hover), 탭 클릭, 정보 패널 슬라이드인 시 세련된 이징 효과(Ease-in-out)를 가미하여 소프트웨어의 시각 완성도를 수배 이상 격상시킵니다. (렌더링 저해나 멀미를 유발하는 현란하고 과장된 3D나 과도한 전환은 철저히 배제합니다.)

## 8. 데이터 사용 규칙
*   **정적 데이터의 승리**: SQLite DB 파일에 물리 소켓 커넥션을 여는 불필요하고 무거운 기법 대신, 클라이언트 메모리에서 즉시 동작하는 정적 JSON 및 CSV 데이터를 채용합니다.
*   **클라이언트 비동기 로드**: 페이지 진입 시 로컬 데이터 셋을 `fetch` API 기반으로 부드럽고 가볍게 비동기 해석 로드합니다.
*   **데이터 에러 바운더리 핸들링**: 정적 데이터 로딩 실패 및 파일 미탐지 등의 비정상 상태 진입 시, 하얀 화면(White-out)으로 침묵하지 않고 "정적 리소스를 로딩할 수 없습니다. 로컬 data/ 폴더 및 파일명을 확인해 주십시오" 등의 수려한 경고 패널이 표시되어 시연을 지켜보는 심사위원이 시스템의 예외 안전 설계 수준을 인지하게 만듭니다.
*   **스키마 정합성 보존**: 모의 데이터셋 내부에 선언된 모든 객체 키(Object Key)와 필드 속성은 반드시 데이터베이스 설계서([context/03_product_database_schema.md](file:///home/jumasi/risk_hunter/context/03_product_database_schema.md))에 공식 설계 및 기재된 스키마 컬럼명 명세와 100% 철저히 통일시킵니다.

## 10. SQL-like Data Explorer 규칙
*   **쿼리 파서의 전격 배제**: 브라우저의 샌드박스 안정성을 파괴하고 연산 지연을 촉발하는 SQLite 쿼리 해석 구문(SQL Parser)을 직접 만드는 일체의 위험한 시도를 완벽히 불허합니다.
*   **템플릿 매핑 기법 채용**: 자주 쿼리하는 필수 통계 SELECT 문을 원클릭으로 선택하면 에디터 패널에 실제 미려한 SQL 문법이 로딩되고, 실행 버튼 클릭 시 자바스크립트로 사전 매핑된 JSON 쿼리 결과 뷰 데이터를 고속 탐색하여 데이터프레임으로 하단에 안전하고 빠르게 뿌려주는 구조로 완결합니다.
*   **안전 보안 샌드박스 장착**: 비인도적 데이터 조작이나 수동 입력 도중 마주할 수 있는 데이터 쓰기 및 파괴성 명령어(`INSERT`, `UPDATE`, `DELETE`, `DROP`, `ALTER`, `CREATE`, `REPLACE` 등)의 기입이 포착되면 대소문자 무관하게 정규식 스캔으로 원천 차단하고 즉각 "SELECT 전용 안전 샌드박스 상태입니다. 원천 마스터 데이터의 보존을 위해 쓰기 행위는 전면 금지됩니다"라는 경고 모달 창을 시각적으로 강하게 인서트해 보안성을 강조합니다.

## 11. 작업 결과 보고 규칙

Gemini 에이전트는 한 단계의 작업이나 페이즈 개발을 매듭지을 때, 사용자를 안심시키고 정합성을 한눈에 판단하기 위해 다음의 **표준 양식 6대 골격**에 맞추어 마감 보고서를 정형화하여 출력 및 전송해야 합니다:

1.  **생성/수정한 파일**: (작업에서 수정/추가된 모든 물리 파일들의 상대/절대 경로 리스트 기재)
2.  **변경 요약**: (기획 및 사양서의 수치 공식을 어떻게 자바스크립트에 준수하여 구현했는지와 디자인 시스템 컬러 연계의 맥락 요약)
3.  **실행 방법**: (브라우저에서 에러 없이 index.html을 더블클릭 및 Live Server로 즉시 로딩해 기동하는 구체적 절차 기술)
4.  **확인 방법**: (특정 버튼을 입력하고 마우스를 올릴 시, 크롬 개발자 도구 콘솔의 예외 에러 유무 확인 방법과 예상 정상 렌더링 확인 가이드 명시)
5.  **현재 한계**: (정적 MVP 범위에 따라 해당 페이즈에서 임베딩되거나 모의 처리된 한계 조건들의 솔직한 기재)
6.  **다음 추천 Phase**: (프로젝트 진행을 위해 연동해야 할 최적의 다음 마일스톤 페이즈 공식 권장)

## 12. 금지 사항
*   **단계 이탈 및 동시 다량 구현**: 현재 활성화된 타겟 Phase 외에 다른 Phase에 배정된 비즈니스 가중치 로직이나 레이아웃을 독단적으로 미리 코딩하거나, 무관한 파일에 변경(Write)을 실행하는 행위.
*   **승인되지 않은 프레임워크/CSS 사용**: React, Vue, Svelte, Angular 및 Tailwind CSS 등 명시적으로 승인되지 않은 일체의 라이브러리, 패키지 및 프레임워크의 무단 사용 및 프로젝트 침투.
*   **백엔드 미들웨어 서버 구축**: Express, Flask, Fast API, Django, NestJS, Python 등 별도의 웹 백엔드 런타임 서버 및 동적 데이터 전송용 소켓 모듈 구축 금지.
*   **인프라 빌드/번들 체인 도입**: Webpack, Vite, Turbopack, Babel, Rollup, Sass 컴파일 등 Node.js 패키지 종속성을 유발하는 빌드 파이프라인 도입 금지.
*   **실제 물리 데이터베이스 연동**: 실시간 SQLite, MySQL 등 물리 이진 데이터베이스 서버 연동 및 브라우저 WASM 단의 `sqlite3.js`/`SQL.js` 직접 디코딩 조회 기법 사용 금지.
*   **불필요한 기능(과잉 엔지니어링) 구현**: 로그인/회원가입/세션 보존 등의 인증 보안 모듈, 완전한 물리 RBAC 제어 엔진 구축, 신규 PDF/DOCX 실물 파일을 현장에서 기계 파싱하는 파이프라인 구축 등 해커톤 MVP 범위를 초과하는 과잉 엔지니어링 일체 금지.
*   **로컬 파일 watchdog 스크립트 가동**: 실시간 규격서 폴더 등의 물리 변화를 감시하는 watchdog 데몬의 프론트엔드 연동 행위 금지.
*   **원천 데이터 파괴 및 무단 쓰기**: 클라이언트 메모리를 완전히 벗어나 물리 하드디스크의 마스터 데이터를 직접 훼손하거나 수정하는 모듈 탑재 금지. SQL 콘솔에서 안전 SELECT 샌드박스를 우회하는 쓰기 명령어(`INSERT`, `UPDATE`, `DELETE`, `DROP` 등) 실행 차단 우회 행위.
*   **과도한 비동기/동적 복잡성**: 과도한 다중 Promise 중첩, 복잡한 드래그 앤 드롭 등 시연 당일의 오동작 확률을 높이는 예측 불가하고 무거운 연출의 무단 도입.
*   **에러 발생 시 무대책 White-out 방치**: API Key 미탐지, 오프라인 차단, 혹은 fetch 실패 등의 상황에서 화면 전체가 하얗게 멈추는 예외 처리 미비 방지 (반드시 우아한 에러 바운더리 폴백 화면 노출).

## 13. 기본 응답 방식
*   사용자의 프롬프트 명령을 받는 즉시 가장 먼저 **"참조한 핵심 기획 문서명" 및 "현재 페이즈 목표"**를 정중하고 일목요연하게 2-3줄 이내로 밝혀 기대 정합성을 싱크합니다.
*   수정할 대상 파일 경로와 빌드외 영역인 placeholder 한계를 명시해 둡니다.
*   코딩 이행 및 자체 마우스 호버 검교정이 완료된 후, 상위 **`11. 작업 결과 보고 규칙`**의 형식을 그대로 수렴한 신뢰성 높은 결과 보고를 마침표 삼아 전송합니다.
*   개발 중 설계의 맹점이나 모호한 예외 사양을 포착하더라도 에이전트의 독자적 상상으로 복잡한 객체 아키텍처를 확장하여 설계하지 않으며, 철저히 정적 해커톤 MVP 규격에 부합하도록 가장 가볍고 미학적이며 작동 안정성이 보장되는 "가장 단순하고 견고한 대안"으로 보정 처리합니다.
